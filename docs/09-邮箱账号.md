# 邮箱账号

OctoManger 把邮箱账号独立成专用资源，和普通 `accounts` 分开管理。

当前内置重点是 Outlook / Microsoft Graph 相关能力：

- OAuth 手动接入 Outlook 邮箱
- Graph Refresh Token 批量导入
- 读取邮箱目录、邮件列表、邮件详情
- 为模块提供“读取最新邮件”的内部 API

## 资源模型

接口前缀：

```text
/api/v1/email/accounts
```

核心字段：

| 字段 | 说明 |
| --- | --- |
| `address` | 邮箱地址，唯一 |
| `provider` | 例如 `outlook`、`gmail`、`custom` |
| `status` | `0` 待验证，`1` 已验证 |
| `graph_config` | 写入用的 Graph / IMAP 风格配置 |
| `graph_summary` | 读取接口返回的脱敏摘要 |

注意：

- 列表和详情接口不会返回原始 `graph_config` 中的敏感值
- 返回的是 `graph_summary`，只告诉你哪些字段存在、邮箱目录是什么、token 是否存在等摘要信息

## Outlook OAuth 配置

控制台邮箱页依赖一个系统配置键：

```text
outlook_oauth_config
```

常见字段：

- `clientId`
- `clientSecret`
- `tenant`
- `redirectUri`
- `scope`
- `mailbox`
- `graphBaseURL`

### 回调地址要和当前页面同源

如果你通过后端静态托管访问控制台：

- 常见回调地址：`https://localhost:8080/oauth/callback`

如果你通过 `bun dev` 访问前端开发服务器：

- 常见回调地址：`http://localhost:5173/oauth/callback`

邮箱页面会校验 `redirectUri` 的 origin 是否与当前页面一致，不一致会直接阻止发起 OAuth。

## 手动添加 Outlook 账号

控制台路径：

- `Email Accounts -> Outlook -> 手动添加`

大致流程：

1. 在设置里填好 Outlook OAuth 配置
2. 输入邮箱地址
3. 页面打开 Microsoft 授权窗口
4. 回调到 `/oauth/callback`
5. 前端把授权码换成 token
6. 后端创建邮箱账号记录

写入的 `graph_config` 典型包含：

- `auth_method=graph_oauth2`
- `username`
- `client_id`
- `client_secret`
- `refresh_token`
- `access_token`
- `token_expires_at`
- `tenant`
- `scope`
- `token_url`
- `graph_base_url`
- `mailbox`

## Graph 批量导入

接口：

- `POST /api/v1/email/accounts/batch-import-graph`

这是异步任务，会：

1. 解析文本内容
2. 对地址做去重
3. 跳过数据库中已存在的邮箱
4. 后台刷新 Graph Token
5. 生成标准化 `graph_config`
6. 批量创建邮箱账号

支持三种行格式：

```text
email----refresh_token
email----client_id----refresh_token
email----password----client_id----refresh_token
```

说明：

- 第二种和第三种都会优先使用行内 `client_id`
- 第一种会回退到请求里的 `default_client_id`
- 第三种里的 `password` 当前仅作兼容占位，导入流程本身不会使用它

典型请求：

```json
{
  "content": "user@outlook.com----refresh_token",
  "default_client_id": "dbc8e03a-b00c-46bd-ae65-b683e7707cb0",
  "tenant": "consumers",
  "scope": [
    "offline_access",
    "openid",
    "profile",
    "email",
    "https://graph.microsoft.com/Mail.Read"
  ],
  "mailbox": "INBOX",
  "graph_base_url": "https://graph.microsoft.com/v1.0",
  "status": 0
}
```

## 批量“注册”邮箱账号

接口：

- `POST /api/v1/email/accounts/batch-register`

名字叫“批量注册”，但当前内置实现要明确区分：

- 它不会去 Outlook / Gmail 官方站点帮你真实注册外部邮箱
- 它会异步生成邮箱地址和默认 `graph_config`
- 然后在 OctoManger 数据库里创建邮箱账号记录

也就是说，它更准确地说是“批量生成邮箱账号记录”。

请求示例：

```json
{
  "provider": "outlook",
  "count": 10,
  "prefix": "seed",
  "domain": "outlook.com",
  "start_index": 1,
  "status": 0,
  "graph_defaults": {
    "client_id": "your-client-id"
  }
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `provider` | 只影响记录的 provider 值 |
| `count` | 1 到 200 |
| `prefix` | 为空时默认 `mail` |
| `domain` | 建议必填，否则生成的地址通常无效 |
| `start_index` | 为空或小于等于 0 时按 1 起算 |
| `status` | `0` 或 `1` |
| `options.mailbox` | 可覆盖默认邮箱目录，默认 `INBOX` |
| `options.graph_config` | 生成阶段附加到默认配置上的字段 |
| `graph_defaults` | 在真正创建记录时作为高优先级默认值写入 |

## 邮件读取能力

接口：

- `GET /api/v1/email/accounts/{id}/mailboxes`
- `GET /api/v1/email/accounts/{id}/messages`
- `GET /api/v1/email/accounts/{id}/messages/latest`
- `GET /api/v1/email/accounts/{id}/messages/{messageId}`

还有两个预览接口：

- `POST /api/v1/email/accounts/preview/mailboxes`
- `POST /api/v1/email/accounts/preview/messages/latest`

预览接口适合在“还没保存邮箱账号记录”时先测试配置是否能连通。

## 验证与批量操作

支持的邮箱专用批量接口：

- `POST /api/v1/email/accounts/batch-delete`
- `POST /api/v1/email/accounts/batch-verify`

单个验证接口：

- `POST /api/v1/email/accounts/{id}:verify`

这里的“验证”只是把 `status` 标记成 `1`，不会自动发起外部邮箱连通性探测。

## 给模块提供邮件能力

OctoModule 不需要自己拼业务邮箱 API，只要使用 `octo.py` 的内部客户端即可：

```python
client = octo.from_context(request.get("context") or {})
message = client.get_latest_email(42, "INBOX")
```

这类内部调用最终走的是：

```text
/api/v1/octo-modules/internal/email/accounts/{id}/messages/latest
```

## 常见问题

### 为什么详情接口看不到完整 `graph_config`

因为后端只返回脱敏后的 `graph_summary`，不回传真实 token 和 secret。

### 为什么 OAuth 配置明明填了，却无法弹窗完成添加

优先检查：

- `redirectUri` 是否和当前页面同源
- `scope` 里是否包含 `offline_access`
- 本地开发时是否把默认 `http://localhost:8080/oauth/callback` 改成了 `http://localhost:5173/oauth/callback`

### 为什么批量注册后无法真实收信

因为当前内置“批量注册”只是在 OctoManger 里创建邮箱记录，并不会替你在外部邮箱服务商完成真实开户。
